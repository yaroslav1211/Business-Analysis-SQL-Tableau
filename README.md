Ось варіант професійного README.md українською мовою, адаптований під твій SQL-код. Він структурований так, щоб одразу показати рекрутеру твою логіку аналізу та рівень володіння інструментами.

Аналітичний проєкт: Аналіз популярності товарів та факторів впливу (SQL)
📌 Огляд проєкту
Цей проєкт присвячений дослідженню факторів, що впливають на успішність товарів на маркетплейсі. Використовуючи набір даних olist_store, я провів шлях від визначення цільових метрик до сегментації продуктів на основі їхньої ефективності.

Основна мета: Виявити закономірності між ціною, швидкістю доставки, рейтингом товарів та кількістю замовлень.

🛠 Технологічний стек
Інструмент аналізу: SQL (BigQuery).

Ключові методи: CTE (Common Table Expressions), Window Functions, складні JOIN-операції, агрегація даних, CASE-логіка для сегментації.

📂 Структура аналізу (SQL скрипти)
Проєкт розділений на 4 логічні блоки, кожен з яких вирішує конкретне бізнес-завдання:

1. Визначення цільової метрики популярності
Завдання: Побудувати рейтинг топ-10 найпопулярніших товарів за кількістю успішних замовлень.

Технічна особливість: Виключення скасованих (canceled) та недоступних (unavailable) замовлень для чистоти аналітики.

2. Аналіз факторів впливу
Завдання: Візуалізувати, як ціна, рейтинг та термін доставки варіюються в залежності від категорії товару.

Метрики: Середній чек, середній бал відгуків та середня тривалість доставки в днях (DATE_DIFF).

3. Пошук аномалій та закономірностей
Завдання: Дослідити гіпотези (наприклад, чи впливає висока ціна на якість відгуків та як швидкість доставки корелює з кількістю замовлень).

Метод: Класифікація товарів за ціновими категоріями (Low, Medium, Big) та швидкістю доставки (Fast, Medium, Low).

4. Сегментація продуктів
Завдання: Розділити базу товарів на стратегічні сегменти для прийняття бізнес-рішень.

Визначені сегменти:

Good-products: Оптимальне поєднання ціни (<60), швидкої доставки (<8 днів) та високого рейтингу (>4).

Bad products: Товари з низьким рейтингом або критично довгою доставкою.

Common markets: Середньостатистичні товари платформи.

🚀 Як використовувати
Файл курсова.sql містить повний код усіх чотирьох етапів аналізу.

Результати запитів можуть бути використані як база для побудови дашбордів у Tableau або Looker Studio (що я і зробив у суміжній частині проєкту).
